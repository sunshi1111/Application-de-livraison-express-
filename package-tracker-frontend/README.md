# 包裹追踪系统前端

这是一个基于 React 的包裹追踪系统前端界面，用于查询包裹位置和显示运输路径。该系统模拟了您的 Python 包裹管理系统的核心功能。

## 功能特性

### 🔍 包裹查询
- 通过包裹 ID 搜索包裹信息
- 查看包裹详细信息（类型、状态、路线等）
- 显示包裹运输历史记录
- 浏览所有包裹列表

### 🗺️ 路径可视化
- 交互式网络图显示站点和中心
- 可视化包裹的最优路径
- 区分不同类型的连接（航线、高速路、道路）
- 动态路径动画和高亮显示

### 📊 智能路径规划
- 快递包裹：优先最短时间路径（Dijkstra算法）
- 标准包裹：优先最低成本路径
- 实时路径计算和优化
- 路径详细信息（时间、费用、距离）

### 📈 数据统计
- 站点和中心数量统计
- 包裹类型分布
- 连接路径统计
- 实时系统状态

## 技术栈

- **前端框架**: React 18
- **数据可视化**: D3.js
- **路径算法**: Dijkstra 算法实现
- **样式**: CSS3 + 响应式设计
- **数据生成**: JavaScript 模拟数据生成器

## 安装和运行

### 前提条件
- Node.js 14.0 或更高版本
- npm 或 yarn 包管理器

### 安装依赖
```bash
cd package-tracker-frontend
npm install
```

### 启动开发服务器
```bash
npm start
```

应用将在 `http://localhost:3000` 启动。

### 构建生产版本
```bash
npm run build
```

## 功能说明

### 包裹管理
- **包裹类型**: 
  - 标准包裹 (70%): 优化成本路径
  - 快递包裹 (30%): 优化时间路径
- **包裹状态**: 已创建、运输中、处理中、已送达、延误

### 网络结构
- **配送中心**: 高吞吐量节点，用于航线连接
- **配送站点**: 本地配送节点，连接到最近的中心
- **连接类型**:
  - 航线: 中心到中心的快速连接
  - 高速路: 中心到站点的连接
  -道路: 站点间的本地连接

### 路径算法
基于您的 Python 代码实现的路径计算：
- 使用 Dijkstra 算法寻找最优路径
- 根据包裹类型选择优化目标（时间 vs 成本）
- 支持动态负载调整和路径重计算

## 使用说明

1. **查询包裹**: 在搜索框中输入包裹 ID 或关键词
2. **查看路径**: 选择包裹后系统会自动计算和显示最优路径
3. **交互探索**: 点击地图上的节点查看详细信息
4. **切换显示**: 使用控制面板切换连接线的显示
5. **重新生成**: 点击"重新生成数据"按钮获取新的模拟数据

## 响应式设计

界面支持多种设备尺寸：
- 桌面端: 双列布局，完整功能
- 平板端: 自适应单列布局
- 手机端: 优化的触摸界面

## 与 Python 后端的对应关系

该前端系统模拟了您的 Python 代码的核心功能：
- `DataGenerator.js` → `data_gen()` 函数
- `PathCalculator.js` → Bellman-Ford 算法实现
- `Package, Node, Route` → 对应的 Python 类
- 路径优化逻辑 → `find_shortest_time_path` 和 `find_lowest_cost_path`
